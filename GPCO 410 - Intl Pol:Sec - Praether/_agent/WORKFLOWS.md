# WORKFLOWS.md

## 1) Weekly reading workflow
1. Identify required readings for the week.
2. Create or update a note in `READING_NOTES/`.
3. Capture for each reading:
   - thesis
   - core concepts
   - mechanism/argument
   - evidence/examples
   - relevance to international politics/security
   - relation to prior theories
4. Add carry-forward insights to `COURSE_MEMORY.md` and `THEORIES.md`.

## 2) Post-lecture synthesis workflow
1. Write a short weekly synthesis note in `WEEKLY_SYNTHESIS/`.
2. Record:
   - what was covered
   - what the instructor emphasized
   - what changed in understanding
   - open questions
3. Update theory entries if a framework was refined, challenged, or connected to a new case.

## 3) Assignment-prep workflow
1. Check `ASSIGNMENTS.md`.
2. Identify relevant theories/frameworks.
3. Pull supporting reading notes and weekly syntheses.
4. Draft argument options before writing full prose.
5. Record what worked so future assignments get easier.

## 4) Exam/study-guide workflow
1. Convert `THEORIES.md` into compact comparison sheets.
2. Group theories by:
   - assumptions
   - unit of analysis
   - causal mechanism
   - policy/security implications
3. Build likely compare/contrast prompts.
4. Add real-world examples for each theory.

## 5) End-of-week maintenance
At the end of each week:
- update theory tracker
- update assignment tracker
- create one synthesis note
- record unresolved confusion
- note any phrases/frames the professor uses repeatedly
